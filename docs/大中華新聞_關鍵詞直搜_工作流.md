流程名字：大中華新聞（關鍵詞直搜）— 懸浮預覽

开始页面：Wisers 搜索结果页（通常由上一板塊搜索结果延续）

执行操作1：準備板塊狀態（ensure_news_session_state）
操作1之后的页面：仍在结果页，session_state 初始化完成

执行操作2：取得關鍵詞預設與內容範圍設定
操作2之后的页面：仍在结果页
细节：
- 读入关键字预设（_get_keyword_presets）
- 读入 include_content（是否搜内文）
- 计算是否周一，决定是否包含 yesterday 周期

执行操作3：设定搜索过滤条件（_apply_search_filters）
操作3之后的页面：仍在结果页
细节：
- 媒體來源篩選：只保留预设媒体标签
- 关键字范围：標題必选，內文按 include_content

执行操作4：执行关键词搜索（run_keyword_search_task）
操作4之后的页面：搜索结果页（通过编辑搜索弹窗完成）
细节：
- 打开「编辑搜索」弹窗并写入关键词
- 点击搜索并等待结果面板就绪
- 等待结果列表可见并完成 Ajax

执行操作5：懸浮抓取预览（scrape_hover_popovers）
操作5之后的页面：搜索结果页
细节：
- 获取可悬浮条目（span[rel='popover-article']）
- 逐条 hover，等待弹窗与预加载结束
- 采集 hover_html 与 hover_text

执行操作6：按字数筛选并补充元数据
操作6之后的页面：搜索结果页
细节：
- 从 hover_text 提取「字数」，筛选 [min_words, max_words]
- 标记 keyword_preset / keyword_preset_index
- 若为周一，补充 day_tag（周日/今日）
- 解析 news_id 与格式化元数据（_build_preview_list_from_raw）

执行操作7：保存预览清单并切换阶段
操作7之后的页面：仍在搜索结果页（浏览器端不变）
细节：
- 写入 session_state：articles_list / pool_dict / sorted_dict / stage
- 保存 Firebase：preview_articles.json、user_final_list.json

结束页面：搜索结果页

输出结果：
- 大中華新聞的懸浮預覽清單
- Firebase：greater_china_keyword_search/preview_articles.json
- Firebase：greater_china_keyword_search/user_final_list.json（初始为空）
